## Doctrine

```
BUS is universal.
MPM is one specialized stream/domain of BUS.
Git is durable memory, audit, and final persistence.
Git is not the fast transport bus.
Direct file runtime is the preferred fast transport layer when available.
```

## Purpose

BUS is the universal transport, operational memory, and inter-agent exchange layer for the yOS cognitive operating system. It provides:

- **Inbox / Workspace / Outbox / Archive** lifecycle for all inter-agent messages, tasks, artifacts, and decisions.
- **Domain-based routing** — each yOS module or project has its own BUS domain.
- **Multiple runtime backends** — direct file (preferred), Manus cloud (probe), Git (fallback), Google Drive, NAS, blob payload.
- **Schema-validated packets** — all BUS messages conform to typed schemas.
- **Cognitive commit convention** — all durable Git-backed BUS events follow the YOS commit convention.

## Relationship to MPM

MPM is a specialized BUS domain. BUS transports MPM packets. MPM executes them.

```
BUS inbox/mpm -> BUS workspace/mpm -> MPM execution runtime -> MPM MPR/report -> BUS outbox/mpm -> BUS archive/mpm
```

MPM canonical ledger and reports remain in `01_BACKBONE/MPM/`. BUS does not replace MPM internals.
