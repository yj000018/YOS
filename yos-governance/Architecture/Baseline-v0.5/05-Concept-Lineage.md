# Y-OS Architecture Baseline v0.5
## 05 — Concept Lineage

**Status:** Working historical map pending raw ARCH transcript verification.

---

# 1. Cockpit lineage

```text
OpenWebUI
→ Y-Menu / yOS Client
→ Manus as current primary cockpit
→ Y-Nexus UI as sovereign target
```

These may be layered roles rather than strict replacements:

- OpenWebUI: technical/self-hosted model cockpit;
- Y-Menu: Y-OS shell/navigation concept;
- Manus: current experience orchestrator;
- Y-Nexus UI: future sovereign system interface.

---

# 2. Orchestration lineage

```text
Y-ORC umbrella
→ CRT + ART + Context Pack + Workflow Planning
→ Y-Nexus Control Plane hosting Y-ORC runtime
```

Working reconciliation:

- Y-Nexus governs jobs, policies, permissions, budget, and state;
- Y-ORC performs orchestration;
- CRT routes cognition;
- ART routes agents, tools, APIs, MCPs, runtimes, and effectors.

---

# 3. Context lineage

```text
Context Builder
→ Y-CTX
→ CCR Runtime
```

Working distinction:

- Y-CTX decides what context is relevant;
- CCR compiles it into portable provider-ready Context Packs.
