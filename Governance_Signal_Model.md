# Governance Signal Model

**Owner:** Chief Architect (Brahma)  
**Status:** Accepted  
**Date:** 2026-06-13  

## 1. Concept
A Governance Signal is an automated alert generated by the Open Loop Engine indicating a deviation from the Operational Value Chain or expected velocity.

## 2. Signal Types

### Type A: Lineage Signals (Structural Integrity)
Generated when the Artifact Lineage is broken.
*   **Missing Parent (L-01):** An artifact was created without a predecessor (except Strategy Briefs). Indicates rogue execution.
*   **Missing Child (L-02):** An artifact was consumed, but no output was produced. Indicates a dropped handoff.
*   **Orphan (L-03):** An artifact lacks a Mission ID. Indicates unclassified work.

### Type B: Velocity Signals (Flow Rate)
Generated when artifacts stagnate in specific states.
*   **Stalled Review (V-01):** Artifact waiting for approval too long.
*   **Stalled Execution (V-02):** Artifact accepted but not yet consumed by the next agent.
*   **Abandoned Draft (V-03):** Artifact started but never submitted for review.
*   **Stalled Rework (V-04):** Artifact rejected but not corrected.

### Type C: Mission Health Signals (Global State)
Generated at the mission level.
*   **Blocked Mission (M-01):** Explicitly marked as blocked.
*   **Missing Learning (M-02):** Mission completed without a Saraswati Learning Report.

## 3. Routing (Future Y-ORC Integration)
Currently, signals are routed to the CEO Briefing. In the future, Y-ORC will route these signals directly to the responsible agents (e.g., pinging a Review Owner for a V-01 signal).
