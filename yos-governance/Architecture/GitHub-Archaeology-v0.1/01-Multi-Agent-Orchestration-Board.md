## Coordination rules

1. Every finding must retain repository, path or commit, date, author when available, and confidence.
2. A concept and its implementation are tracked separately.
3. Historical value is not erased by supersession.
4. Small repositories receive the same initial triage as large repositories.
5. No deletion, merge, archival change or canonical promotion occurs without explicit human approval.
6. Parallel lanes may discover the same artifact; deduplication happens in the consolidation spine, not by suppressing discovery.
7. Contradictions are preserved until resolved by evidence.
8. Repository names do not determine ontological status.
9. KOSMOS / Œuvre, Y-OS core, and vertical embodiments remain distinct but cross-linked.
10. Chronicle-worthy emergence events are recorded during the investigation, not reconstructed only afterward.

## Consolidation spine

```text
raw GitHub evidence
→ repository dossier
→ dated evidence atom
→ concept lineage
→ contradiction / supersession analysis
→ reusable asset candidate
→ architecture implication
→ canon delta candidate
→ human review
```
