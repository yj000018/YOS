# YARP ID and Correlation Policy

> **Version:** v1.0.0
> **Status:** candidate

---

## ID Taxonomy

| ID | Format | Scope | Owner | Description |
|---|---|---|---|---|
| `yarp_message_id` | `YARP-MSG-{uuid4}` | global | sender | Unique per message envelope |
| `correlation_id` | `YARP-CORR-{uuid4}` | session | orchestrator | Links EXECUTE_MP → ACK → RESULT |
| `conversation_id` | `YARP-CONV-{uuid4}` | session | orchestrator | Groups all messages in a session |
| `task_id` | `{transport-specific}` | transport | executor | Manus API task ID or BUS claim ID |
| `mp_id` | `MPM-{YYYYMMDD}-{SLUG}` | global | A&G | Canonical MP identifier |
| `mpr_id` | `MPM-{YYYYMMDD}-{SLUG}-REPORT` | global | Manus | Canonical MPR identifier |
| `artifact_id` | `YARP-ART-{uuid4}` | global | creator | Unique per artifact |
| `agent_id` | `agent-{name}-{role}` | global | yOS | Stable agent identifier |
| `transport_id` | `{transport-name}` | global | yOS | Transport adapter identifier |
| `attempt_id` | `YARP-ATT-{uuid4}-{NNN}` | correlation | sender | Unique per retry attempt |
| `session_id` | `YARP-SESSION-{uuid4}` | session | orchestrator | Session lifecycle identifier |

## ID Generation Rules

```
yarp_message_id:  UUID4, generated by sender at message creation time
correlation_id:   UUID4, generated by orchestrator at EXECUTE_MP creation time
conversation_id:  UUID4, generated at session initiation
task_id:          Assigned by executor (Manus API returns task_id in ACK)
mp_id:            Assigned by A&G at MP creation time (canonical format)
mpr_id:           Derived from mp_id: {mp_id}-REPORT
artifact_id:      UUID4, generated by creator
agent_id:         Stable, human-readable, assigned at agent registration
transport_id:     Stable, human-readable, assigned at adapter registration
attempt_id:       UUID4 + 3-digit sequence (001, 002, 003...)
session_id:       UUID4, generated at session initiation
```

## Correlation Chain

```
orchestrator creates:
  correlation_id = YARP-CORR-{uuid4}
  conversation_id = YARP-CONV-{uuid4}

EXECUTE_MP carries:
  correlation_id
  conversation_id
  mp_id

executor returns ACK with:
  ack_for_message_id = EXECUTE_MP.yarp_message_id
  task_id = assigned by executor

executor returns RESULT with:
  correlation_id = same as EXECUTE_MP
  mp_id = same as EXECUTE_MP
  task_id = same as ACK

orchestrator matches RESULT to EXECUTE_MP via:
  correlation_id (primary)
  mp_id (secondary)
  task_id (tertiary)
```

## Idempotency Rules

### EXECUTE_MP Idempotency

```
Key: mp_id
Rule: If executor receives EXECUTE_MP with mp_id already in:
  - running/claimed/acknowledged state → return ACK with existing task_id, do NOT re-execute
  - completed state → return existing RESULT (idempotent response)
  - failed/timed_out state + new attempt_id → accept as retry, start fresh
  - queued state → accept, start execution
```

### ACK Idempotency

```
Key: ack_for_message_id
Rule: If orchestrator receives duplicate ACK → ignore silently
```

### RESULT Idempotency

```
Key: correlation_id
Rule: If orchestrator receives duplicate RESULT → process first, ignore subsequent
```

## Retry Attempt Tracking

```
attempt_id format: YARP-ATT-{base_uuid}-{NNN}
  base_uuid: same for all attempts of the same EXECUTE_MP
  NNN: 001, 002, 003...

Example:
  Attempt 1: YARP-ATT-f3a8b2c1-001
  Attempt 2: YARP-ATT-f3a8b2c1-002
  Attempt 3: YARP-ATT-f3a8b2c1-003
```

## Audit Trail Requirements

Every YARP message MUST be logged with:

```json
{
  "logged_at": "ISO8601",
  "yarp_message_id": "...",
  "message_type": "...",
  "correlation_id": "...",
  "sender_id": "...",
  "receiver_id": "...",
  "transport_id": "...",
  "attempt_number": 1,
  "outcome": "sent|received|acked|nacked|timeout|error"
}
```

Audit log path: `01_BACKBONE/YARP/04_GOVERNANCE/audit/YARP-AUDIT-{YYYYMMDD}.jsonl`

## MPM Ledger Integration

YARP correlation_id maps to MPM ledger entries:

```
mp-ledger.json entry:
  mp_id → YARP mp_id
  status: running → YARP state: running/acknowledged/claimed
  status: executed → YARP state: completed
  commit → YARP RESULT.commit
  canonical_mpr_path → YARP RESULT.mpr_path
```
