# Capability Roadmap: Lead Developer (Hanuman)

**Status:** Official | **Date:** 2026-06-12 | **Owner:** CODO (Saraswati)

## Concept
Hanuman is the universal build engine of Y-OS. While currently focused on code and scripts, Hanuman's capabilities must expand to construct any artifact designed by Brahma.

## Current Capabilities (v1 - Active)

1. **Python Scripting:** Building standalone scripts, automation tools, and data processing pipelines.
2. **MCP Integration:** Utilizing Model Context Protocol servers to interact with external systems (e.g., Notion, GitHub).
3. **Shell Automation:** Executing complex sandbox commands to manipulate files, install dependencies, and run utilities.
4. **Markdown Generation:** Compiling complex, structured documents based on templates.

## Near-Term Expansion (v1.5 - Next 3 Months)

1. **Web Application Build:** Consuming web architecture packages to scaffold and deploy full web apps (React/Next.js) via the `webapp-factory` pipeline.
2. **Data Visualization:** Building complex charts, graphs, and Excel models based on data architecture packages.
3. **API Construction:** Building and deploying custom REST APIs (FastAPI/Flask) within the sandbox or external servers.

## Long-Term Vision (v2.0 - Universal Builder)

1. **Infrastructure as Code (IaC):** Writing Terraform or Ansible scripts to provision cloud resources based on Brahma's infrastructure designs.
2. **Media Generation:** Consuming content architectures to orchestrate video, audio, and image generation pipelines.
3. **Agent Configuration:** Building and configuring new sub-agents based on organizational designs provided by CODO/Brahma.
