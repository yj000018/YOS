## OG-003 — The Constitution Pattern

Every major module received a Constitution before it received an implementation.

Order of operations:
1. Constitution (immutable doctrine)
2. Spec (formal specification)
3. Schemas (machine-readable contracts)
4. Adapters (implementation bridges)
5. Tools (CLI utilities)
6. Tests (validation)

This is not a software development pattern. It is a governance pattern.
The Constitution is the soul; the implementation is the body.

## OG-005 — The Empirical Method

yOS was not designed top-down. It was discovered empirically.

Each gate was a probe:
- DIRECT-FILE-RUNTIME-PROBE-GATE → discovered /home/ubuntu/ persistence
- MANUS-WORKSPACE-PROBE-GATE → classified manus_workspace as candidate
- CONNECTIVITY-CENSUS-GATE → mapped all Manus connectivity mechanisms

The architecture emerged from empirical probes, not from blueprints.
