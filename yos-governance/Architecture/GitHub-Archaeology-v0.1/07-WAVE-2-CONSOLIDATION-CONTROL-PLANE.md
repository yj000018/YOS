# Wave 2 — Consolidation Control Plane

**Program:** GitHub Account Archaeology v0.1  
**Branch:** `agent/github-account-archaeology-wave1`  
**Status:** ACTIVE  
**Primary constraint:** consolidate before expanding.  
**Execution model:** coordinated parallel evidence lanes with one shared synthesis surface.

## 1. Mission

Wave 2 does not create a new KAP, archaeology framework, memory architecture, or Manus workflow.

It reconstructs exactly what already exists, what was actually executed, what remains partial, and what can be reused without paying Manus to rediscover it.

The governing sequence is:

```text
Recover existing work
→ establish completion boundaries
→ reconcile counts and terminology
→ identify reusable capabilities
→ map gaps
→ prepare a minimal Manus continuation package
```

## 2. Anti-fragmentation rule

All parallel lanes feed this single control plane and the existing Meta-Archaeology Register.

No new independent framework, registry family, ontology, or agent taxonomy is authorized unless the evidence proves that an essential mechanism is missing.

Every discovered artifact is classified as one of:

- `REUSE_AS_IS`
- `REUSE_WITH_DESTINATION_UPDATE`
- `RECONCILE_WITH_LATER_DOCTRINE`
- `HISTORICAL_ONLY`
- `PARTIAL_REQUIRES_COMPLETION`
- `BLOCKED_UNTIL_MANUS`
- `MISSING_CONFIRMED`

## 5. Consolidation-first execution policy

1. Search existing registries, commits, reports, skills and scripts before proposing anything new.
2. Prefer updating the Meta-Archaeology Register and this control plane over creating additional standalone documents.
3. Keep provenance and date attached to every conclusion.
4. Separate factual recovery from architectural judgement.
5. Mark uncertainty explicitly.
6. Do not call a corpus complete without naming the completed pipeline stage.
7. Do not use Manus later for work already reproducible from Git.
8. Do not use ChatGPT now to imitate inaccessible Manus-only operations.
9. No destructive repository or canon action during Wave 2.

## 6. Wave 2 exit criteria

Wave 2 is complete only when:

- the 194/363/316/593 counts are semantically defined;
- Manus metadata, body, output and factsheet coverage are distinguished;
- prior KAP/Manus agents, skills, scripts and gates are inventoried;
- every reusable mechanism has a current reuse decision;

- source-authority transitions are dated;
- the remaining Manus-only work is smaller, bounded and non-redundant;
- one continuation prompt/package can be given to a clean Manus session without asking it to rediscover KAP.
