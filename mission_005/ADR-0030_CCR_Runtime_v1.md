# ADR-0030 — CCR Runtime v1: Context Continuity Engine

**Status:** Proposed (MISSION-005 candidate)  
**Date:** 2026-06-13  
**Deciders:** Krishna (strategy), Brahma (architecture), Lakshmi (governance)  
**Supersedes:** ADR-0027 (Context Continuity baseline)

---

## Context

MISSION-005 produced a complete architectural specification for CCR Runtime v1. Lakshmi confirmed COMPLIANT on all 5 constitutional principles. Ganesha recommended ADOPT.

## Decision

Y-OS adopts CCR Runtime v1 as the official Context Continuity Engine with the following canonical architecture:

**7 components:** Artifact Retriever, Lineage Traverser, Context Selector, Compression Engine, Context Pack Generator, Mission Continuity Manager, Governance Hook.

**3 compression levels:** FULL (~4000 tokens), COMPRESSED (~2000 tokens), MINIMAL (~800 tokens).

**Provider portability:** Context Packs are YAML-serialized, provider-agnostic, and self-contained.

**Governance:** Lakshmi observes every CCR execution via the Governance Hook. Human override always available.

## Consequences

- CCR becomes a mandatory step in the Y-ORC → ART → CCR → CRT pipeline.
- All workers receive Context Packs, never raw conversation history.
- Context Pack Schema v2 replaces v1.
- CCR execution is logged and auditable.
