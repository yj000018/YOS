# Y-OS Org Map v2 (Master Document)

**Status:** Official | **Date:** 2026-06-12 | **Owner:** CEO (Yannick) | **Custodian:** CODO (Saraswati)

## Purpose
This document is the canonical single source of truth for the Y-OS v1 architecture. It consolidates all validated roles, layers, workflows, and artifacts into a single cohesive model. A new team member must be able to understand the entirety of Y-OS by reading this document.

## 1. Executive Structure (The Roles)
Y-OS operates through a formalized executive team. Roles are organizational interfaces, not specific software models.

- **CEO (Yannick):** The human operator. Defines ultimate intent, validates strategy, and provides final authorization.
- **CSO (Krishna):** Defines Strategy. Answers *What* and *Why*.
- **COO (Ganesha):** Defines Execution. Answers *When* and *Who*. Owns the delivery of the final artifact.
- **Chief Architect (Brahma):** Defines Design. Answers *How*. Creates the structural blueprint.
- **Lead Developer (Hanuman):** Executes the Build. Translates blueprints into reality.
- **CODO (Saraswati):** Defines Organizational Evolution. Improves the system that solves the problems.
- **ECO (Lakshmi):** Defines Visibility. Maintains the Executive Dashboard and Open Loops Register.

## 2. Layer Architecture (The Enduring Structure)
While roles are transient interfaces, layers are the persistent structure of Y-OS.

1. **Executive Layer:** Intent & Visibility (CEO, Krishna, Lakshmi).
2. **Execution Layer:** Orchestration & Delivery (Ganesha).
3. **Design Layer:** Structural Definition (Brahma).
4. **Build Layer:** Implementation (Hanuman).
5. **Evolution Layer:** Governance & Learning (Saraswati).
6. **Artifact Layer:** The immutable system of record.
7. **Capability Layer:** The replaceable compute engines (Manus, LLMs, Tools).
8. **Memory Layer:** The cumulative intelligence (Y-MEM, Notion, Git).

## 3. Operational Value Chain (OVC)
The immutable flow of execution:
`Strategy → Execution Planning → Design → Build → Delivery → Learning`

## 4. The Artifact Layer
Agents do not communicate directly. They communicate via artifacts.
- **Strategy Brief:** Krishna → Ganesha
- **Execution Plan:** Ganesha → Brahma
- **Architecture Package:** Brahma → Hanuman
- **Build Artifact & Report:** Hanuman → Ganesha
- **Delivery Report:** Ganesha → CEO/Krishna
- **Learning Report:** Saraswati → System

## 5. Foundational Principles
- **Law #7:** The organization is a system that must continuously improve.
- **Law #8:** The goal is to improve the organization that solves problems.
- **Law #9:** No agent defines itself; CODO defines agents.
- **Law #10:** The Operational Value Chain is immutable.
- **Law #11 (Continuity):** Agents are transient. Artifacts are persistent. Capabilities are replaceable. Memory is cumulative. The organization survives the replacement of any individual agent.
