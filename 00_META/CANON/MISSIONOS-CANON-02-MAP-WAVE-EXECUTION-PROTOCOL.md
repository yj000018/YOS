# MAP Wave Execution Protocol

**Status:** CANONICAL  
**Owner:** MissionOS  
**Guardian:** ChatGPT  
**Version:** v1.0  
**Adopted:** 2026-07-17

> Do not interrupt Yannick for operational decisions that are reversible.

## Purpose

Maximize productive parallel work while minimizing:

- Git operations
- approval prompts
- connector latency
- cognitive interruptions
- merge conflicts
- context switching

The objective is to keep Yannick in flow.

## Principle

Never use Git as a working memory.

Git is a persistence checkpoint.

Work happens first.

Persistence happens in waves.

## MAP Wave Lifecycle

```text
Mission
  ↓
MAP Contract
  ↓
Parallel Expeditions
  ↓
Return Packages
  ↓
Guardian Review
  ↓
Checkpoint
  ↓
Git
  ↓
Canon
```

## Default Policy

### During execution

NO Git writes.

Agents work independently.

Artifacts remain inside the current MAP Wave.

### Checkpoint 1

When a coherent wave is finished:

ONE grouped Git checkpoint.

Examples:

- Mission Registry
- DNA
- Cockpit
- Guardian
- Bridges

committed together.

### Guardian Phase

Guardian performs:

- consistency review
- contradiction detection
- duplicate detection
- naming validation
- architectural review
- acceptance/rejection

No Git writes required.

### Checkpoint 2

Only after Guardian approval:

ONE integration checkpoint.

This produces:

- Canon updates
- Registry updates
- merged artifacts
- Pull Request

## Git Usage Policy

Git is used only when one of the following is true:

- significant milestone reached
- durable artifact produced
- risk of losing work
- repository state must be inspected
- Guardian approved integration

Otherwise:

NO Git call.

## Connector Optimization

Default target during one MAP Wave:

- Git reads: as few as possible
- Git writes: 1–2 grouped checkpoints maximum

Never:

- many small commits
- many connector confirmations
- many approval interruptions

## MAP Return Package

Each Expedition returns:

- objective
- completed work
- produced artifacts
- unresolved questions
- dependencies
- recommended next action

Guardian integrates these packages.

Expeditions never modify Canon directly.

## Benefits

- dramatically fewer connector prompts
- lower latency
- lower cognitive interruption
- fewer merge conflicts
- cleaner history
- easier Guardian review
- higher throughput

## Canon Rule

Think in MAP Waves.

Not in Git operations.
