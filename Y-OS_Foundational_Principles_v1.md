## 1. Core Thesis

The Y-OS architecture is governed by a singular, overarching thesis that separates execution from state, and state from visibility.

*   **Agents perform work.**
*   **Artifacts preserve state.**
*   **The Registry records truth.**
*   **Lineage preserves causality.**
*   **Lakshmi provides visibility.**
*   **Open Loops generate governance signals.**

Together, these components form the Y-OS Control Plane, elevating the system from a collection of scripts into a governed operating system.

## 2. Foundational Operational Principles

The operational rhythm of Y-OS is defined by six continuous realities:

1.  **Execution is transient.** The act of computing, writing, or designing is ephemeral.
2.  **State is persistent.** The output of execution must be captured as a permanent artifact.
3.  **Truth is registered.** An artifact does not officially exist until it is logged in the Artifact Registry.
4.  **Causality is traceable.** No artifact exists in a vacuum; every artifact has a parent and/or a child.
5.  **Visibility is continuous.** The executive layer must always be able to observe the state of the system.
6.  **Governance is actionable.** Visibility must produce signals (Open Loops) that can be acted upon.

## 3. The Operational Chain

The transformation of intent into governed reality follows a strict, unbroken chain:

```text
Agents
  → produce Artifacts

Artifacts
  → preserve State

Registry
  → records Truth

Lineage
  → preserves Causality

Lakshmi
  → provides Visibility

Open Loops
  → generate Governance Signals

Y-ORC (Future)
  → executes Decisions
```

## 4. The Layer Model

Y-OS is structured as a stack of specialized layers, each serving a distinct architectural purpose.

*   **Execution Layer:** Agents (Capabilities)
*   **State Layer:** Artifacts
*   **Truth Layer:** Registry
*   **Causality Layer:** Lineage
*   **Visibility Layer:** Lakshmi
*   **Governance Layer:** Open Loops
*   **Orchestration Layer:** Y-ORC

## 5. Official Definition of Y-OS

> Y-OS is not a collection of agents.  
> Y-OS is a governed artifact-centric operating system.

## 6. The Persistence Principle

The value of the Control Plane lies in preventing the degradation of information:

*   Without artifacts, agents forget.
*   Without lineage, artifacts become disconnected documents.
*   Without visibility, lineage becomes inert data.
*   Without governance, visibility becomes passive dashboards.

The Control Plane turns static information into operational decisions.
